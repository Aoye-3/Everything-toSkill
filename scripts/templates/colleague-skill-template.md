# 同事技能项目模板

## 模板说明
此模板专门用于转换类似"colleague-skill"的项目，这些项目将人的知识、经验和沟通风格转化为AI可用的技能。

## 项目特点分析

### colleague-skill项目洞察
1. **核心概念**: 将离职同事的知识转化为持续工作的AI技能
2. **数据结构**: 人物设定(meta.json)、工作文档(work.md)、沟通风格(persona.md)
3. **多语言支持**: 多种语言的README文件
4. **实际应用**: 解决知识传承和团队连续性实际问题
5. **情感元素**: 将"冰冷的离别"转化为"温暖的技能"

## 技能结构

### 标准同事技能目录结构
```
colleague-skill/
├── SKILL.md                    # 主技能文档
├── agents/                     # AI代理元数据
│   └── openai.yaml            # OpenAI兼容元数据
├── colleagues/                 # 同事数据目录
│   ├── {colleague_name}/      # 单个同事数据
│   │   ├── meta.json          # 元数据（角色、专长等）
│   │   ├── persona.md         # 人物设定和沟通风格
│   │   ├── work.md            # 工作文档和知识
│   │   ├── examples/          # 工作示例
│   │   └── references/        # 参考文档
│   └── index.json             # 同事索引
├── scripts/                    # 处理脚本
│   ├── import-colleague.py    # 导入同事数据
│   ├── query-colleague.py     # 查询同事知识
│   └── generate-response.py   # 生成响应
├── references/                 # 参考文档
│   ├── data_sources.md        # 数据源说明
│   ├── import_guide.md        # 导入指南
│   └── usage_patterns.md      # 使用模式
└── assets/                    # 资源文件
    ├── templates/             # 数据模板
    └── examples/              # 示例文件
```

## 核心概念

将同事的知识、经验和沟通风格转化为持续可用的AI技能，实现知识传承和团队连续性。

## 快速开始

### 导入同事数据
```bash
python scripts/import-colleague.py --name "同事姓名" --source "数据源"
```

### 查询同事知识
```bash
python scripts/query-colleague.py --colleague "姓名" --query "技术问题"
```

### 生成工作响应
```bash
python scripts/generate-response.py --colleague "姓名" --context "工作场景"
```

## 数据源支持

### 自动采集
- **飞书**: 消息记录、文档、多维表格（API自动采集）
- **钉钉**: 文档、多维表格（浏览器自动化）
- **Slack**: 消息记录（API，免费版限90天）

### 手动导入
- **PDF文档**: 工作文档、报告
- **图片/截图**: 聊天截图、文档截图
- **聊天记录**: 微信、飞书JSON导出
- **邮件**: .eml/.mbox格式
- **Markdown**: 文档文件
- **直接文本**: 粘贴文字内容

## 同事数据结构

### meta.json（元数据）
```json
{
  "name": "同事姓名",
  "role": "职位角色",
  "expertise": ["领域专长1", "领域专长2"],
  "tenure": "在职时间",
  "communication_style": "沟通风格",

  "preferences": {

  }
```

## 使用场景

### 场景1: 知识传承
**问题**: 同事离职，留下大量未文档化知识
**解决方案**: 导入同事的工作数据，创建持续可用的技能
**效果**: 新成员可以"咨询"离职同事的知识

### 场景2: 工作连续性
**问题**: 关键人员暂时离开，工作受阻
**解决方案**: 使用同事技能继续工作
**效果**: 工作不中断，保持一致性

### 场景3: 培训辅助
**问题**: 新员工需要快速上手
**解决方案**: 使用同事技能作为培训资源
**效果**: 加速学习曲线，减少培训时间

### 场景4: 质量保证
**问题**: 需要保持工作质量标准
**解决方案**: 使用同事技能验证工作质量
**效果**: 保持一致性，减少错误
